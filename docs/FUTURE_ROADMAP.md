# MakeSlide 未來功能路線圖

> 本文件整理目前程式分析後，對 MakeSlide 後續可加入之高價值主要功能的建議。重點不是新增零散功能，而是把現有「AI 內容生成、課堂互動、同步播放、測驗、觀測性、版本與匯出」能力串成完整教學閉環。

## 1. 現有產品基礎

MakeSlide 目前已經不只是 PDF 語音簡報工具，而是接近「AI 教學內容生成 + 課堂同步互動 + 簡報播放/編輯」平台。

### 1.1 核心能力

- 支援 PDF、文字、YouTube 三種來源匯入。
- 可自動生成投影片圖片、逐字稿、語音、標題與播放素材。
- 支援播放頁字幕、播放速度、全螢幕、課堂模式與互動模式。
- 支援同步播放、學生提問、AI 回答、即時投票與測驗。
- 支援手寫標註、動畫效果、圖片風格、逐頁重生、批次重生、取消與 rollback。
- 支援 ZIP 匯入/匯出、影片生成、handout PDF、分享連結與 QR code。
- 支援系統設定、LLM/TTS provider、API key、技能/模板雛形、SLA 與觀測性資料。

### 1.2 架構觀察

- 前端以 React + Vite + Tailwind 為主。
- 後端以 Fastify + TypeScript 為主。
- 生成流程以背景 pipeline 與 per-page artifacts 為核心。
- API 已拆成多個 route module，例如上傳、詳情、頁面操作、同步、測驗、匯出入、動畫、圖像素材、run history、slow artifacts、SLA settings、skills 等。
- 目前最有價值的下一步，是讓既有資料產生「教學後續價值」，而不只是完成單次素材生成。

## 2. 高價值主要功能建議

### 2.1 課後學習報告與學生學習分析

這是最建議優先加入的功能。

目前系統已有同步播放、學生提問、投票、測驗、答題紀錄等資料基礎，下一步可以整理為課後報告。

#### 功能內容

- 每位學生參與紀錄：進入時間、答題完成率、提問次數、投票參與率。
- 每份簡報學習成效摘要：平均分數、答錯率最高題目、最常被問的問題。
- 每頁投影片困難度：在哪些頁學生停留、提問、答錯或投票分歧最多。
- AI 自動產出課後建議：哪些概念需要補充、下一堂課可加強什麼。

#### 產品價值

- 把 MakeSlide 從「簡報生成器」提升成「教學閉環工具」。
- 直接利用現有測驗、投票、同步與問答功能。
- 對教師、補教、企業內訓都有明確價值。

#### MVP 建議

- 在每份簡報新增「課後報告」頁。
- 整合 quiz attempts、poll votes、sync questions。
- 第一版先做班級層級摘要，再延伸到個人層級報表。

---

### 2.2 AI 導師 / 學生自學模式

播放頁目前已有 follower 提問與 AI 回答能力，可擴展成自學模式。

#### 功能內容

- 學生播放簡報時可隨時詢問 AI。
- AI 根據目前頁面、講稿、來源 PDF/YouTube 內容回答。
- AI 可產生補充例題、延伸閱讀、重點摘要。
- 可根據學生答錯題目推薦回看頁面。
- 可生成個人化複習清單。

#### 產品價值

- 讓簡報從一次性播放素材變成互動式自學教材。
- 對非同步課程、線上課程與自主學習很有價值。
- 可與現有問答、來源內容、測驗資料高度整合。

#### MVP 建議

- 在播放頁加入「問這一頁」AI 導師面板。
- 回答時引用目前頁 script、source text、figure metadata。
- 測驗提交後自動產生個人化複習建議。

---

### 2.3 成本預估與生成前試算

目前已有 LLM usage 與 estimated cost 顯示，可延伸為生成前預估與預算控制。

#### 功能內容

- 上傳 PDF 或輸入文字後，先估算生成成本。
- 顯示圖片、腳本、語音、動畫各階段預估成本。
- 提供省錢、平衡、高品質模式。
- 支援每月預算、單次任務上限。
- 超過門檻時提示或禁止生成。

#### 產品價值

- AI 生成服務常見痛點是成本不透明。
- 對多使用者部署與教育現場管理非常重要。
- 可延伸現有 observability 與 usage summary。

#### MVP 建議

- 在開始生成前加成本預估 modal。
- 用頁數、文字量、TTS 長度粗估成本。
- 在 Settings 加入預算上限設定。

---

### 2.4 教材知識庫與跨簡報搜尋

目前首頁偏向卡片列表與分類管理，當資料量增加後，需要知識庫與全文搜尋能力。

#### 功能內容

- 跨所有簡報搜尋標題、逐字稿、來源文字、頁面內容。
- 依主題、關鍵字、課程、標籤整理教材。
- 找到特定概念後直接跳到對應投影片頁面。
- AI 問答可跨多份簡報回答。
- 支援「從知識庫挑選數頁組成新簡報」。

#### 產品價值

- 讓 MakeSlide 成為長期教材資產管理工具。
- 有助於教師複用過去教材內容。
- 可延伸至 RAG、教材重組與知識管理。

#### MVP 建議

- 建立簡單全文索引：title、page script、page text。
- 首頁加全域搜尋。
- 搜尋結果顯示簡報、頁碼、片段與跳轉連結。

---

### 2.5 AI 課程包生成

目前系統已能生成投影片、語音、測驗與 handout PDF，可包裝成「一鍵課程包」。

#### 功能內容

- 教師輸入課程目標、對象、時間長度。
- AI 自動生成：
  - 簡報頁面。
  - 講稿。
  - 課堂投票。
  - 測驗題。
  - 課後作業。
  - 學習單 / 講義。
- 支援輸出為 ZIP 或 LMS 匯入格式。

#### 產品價值

- 比單純投影片生成更貼近教師實際工作流。
- 可串接既有生成、測驗、handout、ZIP 匯出等能力。
- 適合教育、內訓、研習與線上課程製作。

#### MVP 建議

- 在播放頁或首頁新增「生成課程包」按鈕。
- 第一版產生講義 PDF + 測驗 + 課後練習 Markdown。
- 後續支援 Moodle、Google Classroom 或 SCORM。

---

### 2.6 手機 / 平板課堂控制器

目前全螢幕播放、同步、投票與手寫已有基礎，可進一步做教師控制器。

#### 功能內容

- 手機作為遙控器控制播放、上一頁、下一頁。
- 手機上看講稿與備註，投影端只顯示投影片。
- 啟動投票、顯示答案、收合問答。
- 雷射筆 / 游標控制。
- 手機手寫同步到投影端。

#### 產品價值

- 對真實教室場景非常實用。
- 可利用現有 sync state、remote cursor、drawing sync 雛形。
- 能大幅提升課堂操作體驗。

#### MVP 建議

- 新增 `/remote/:id` 控制器頁。
- 共用既有 sync API。
- 第一版支援播放控制、頁面切換、投票開關。

---

### 2.7 生成品質評估與自動修復

目前已有失敗重試、slow artifact、SLA、regenerate，可再加入 AI 品質檢查。

#### 功能內容

- 檢查腳本是否太長或太短。
- 檢查語音是否缺失或長度異常。
- 檢查圖片是否與頁面內容不符。
- 檢查字幕與語音是否不同步。
- 檢查動畫是否超出頁面或時間太長。
- 檢查測驗答案是否不唯一或解釋不足。

#### 產品價值

- AI 生成內容常有不穩定問題，自動 QA 可降低人工檢查成本。
- 可直接串接既有 regenerate / rollback 流程。
- 有助於提升整體成品可信度。

#### MVP 建議

- 在生成完成後跑 quality-check stage。
- 在播放頁顯示「品質警告」。
- 提供「一鍵修復此頁」。

---

### 2.8 模板市集與教學風格模板

目前已有圖片風格 template、TTS 設定、skills，可產品化成模板系統。

#### 功能內容

- 簡報模板：學術報告、國中課程、企業內訓、醫學教學、程式課程。
- 圖像風格模板：手繪、白板、科技風、扁平插畫、黑板風。
- 講稿語氣模板：正式、對話式、兒童友善、考前複習。
- 測驗模板：診斷測驗、形成性評量、期末複習。
- 模板匯入、匯出與分享。

#### 產品價值

- 降低使用者 prompt 門檻。
- 可結合既有 skills 與 image style prompt。
- 容易形成社群分享與可複用資產。

#### MVP 建議

- 把現有 skills 擴充成「模板」概念。
- 模板包含 script prompt、image style、quiz prompt、tts preference。
- 首頁建立「從模板建立」入口。

---

### 2.9 協作編輯與版本審閱流程

目前已有分享、可編輯公開模式、版本歷史、頁面重生與 rollback，可延伸為協作流程。

#### 功能內容

- 多人協作編輯投影片腳本、prompt、動畫。
- 留言與批註。
- 修改建議 / 審閱模式。
- 版本差異比較。
- 還原到特定版本。

#### 產品價值

- 適合教師共備、助教協作與團隊教材製作。
- 可結合既有版本歷史路由與分享機制。
- 有助於打造校內教材共備平台。

#### MVP 建議

- 先做單頁腳本 diff 檢視。
- 加入 comment thread，綁定 page number 與 script / prompt / image 區域。
- 再做多人即時編輯或鎖定式編輯。

---

### 2.10 更完整的匯出與平台整合

目前已有 video、handout PDF、ZIP 匯出，可補強外部平台整合。

#### 功能內容

- 匯出 PowerPoint / Google Slides。
- 匯出 SCORM / xAPI，用於 LMS。
- 匯出 H5P 互動內容。
- 匯出 YouTube 影片與字幕檔。
- 匯出逐字稿、SRT / VTT 字幕。

#### 產品價值

- 老師與企業通常已有既定平台，匯出能力會直接影響採用率。
- 系統內已有頁面圖片、語音、字幕時間軸與影片生成，擴充空間大。

#### MVP 建議

- 先做 SRT / VTT 匯出與 PPTX 匯出。
- PPTX 第一版可採「每頁一張圖片 + speaker notes」簡化版本。

## 3. 優先順序建議

若以「使用者價值 / 實作成本 / 與現有架構相容性」綜合排序，建議如下：

1. 課後學習報告與學生學習分析。
2. AI 導師 / 學生自學模式。
3. 成本預估與生成前試算。
4. 教材知識庫與跨簡報搜尋。
5. AI 課程包生成。
6. 手機 / 平板課堂控制器。
7. 生成品質評估與自動修復。
8. 模板市集與教學風格模板。
9. 協作編輯與版本審閱流程。
10. 更完整的匯出與平台整合。

## 4. 建議階段性路線圖

### Phase 1：把互動資料變成教學洞察

- 課後學習報告。
- 測驗、投票、問答統計。
- 每頁困難度與學生參與度分析。

### Phase 2：強化學生端自學價值

- AI 導師 / 問這一頁。
- 答錯題自動推薦回看頁面。
- 個人化複習清單。

### Phase 3：降低使用與營運成本風險

- 生成前成本預估。
- 預算上限。
- 品質檢查與一鍵修復。

### Phase 4：累積長期教材資產

- 全文搜尋。
- 跨簡報知識庫。
- 頁面重組成新簡報。

### Phase 5：擴大課堂與平台整合

- 手機 / 平板課堂控制器。
- PPTX、SRT、VTT、SCORM、H5P 匯出。
- 模板系統與協作審閱。

## 5. 總結

MakeSlide 目前最強的基礎不是單一 AI 生成能力，而是已經同時具備內容生成、播放、互動、測驗、同步、觀測性、版本與匯出。下一階段最有價值的方向，是把這些能力串成教學閉環。

短期最建議優先投入：

1. 課後學習報告與學生學習分析。
2. AI 導師 / 學生自學模式。
3. 成本預估與生成前試算。

這三項能最大化利用目前已實作的同步、測驗、投票、問答、播放與觀測性資料，並同時提升教師端、學生端與系統管理端的價值。
